# Git and runtime data policy

Generated: 2026-06-09

## Goal

Track source code, tests, scripts, and documentation. Do not track runtime data generated by local operation of the attendance system.

## Policy

Track:

- `app/`, `src/`, `services/`, `repositories/`, `vision/`
- `tests/`
- `scripts/`
- `docs/`
- project Markdown documents
- configuration/source files needed to run the app

Do not track:

- `app/attendance.db`
- `*.db`
- `*.sqlite3`
- `data/`
- `data/embeddings/db.pkl`
- `backups/`
- virtual environments and caches

If seed data is needed later, use explicit folders such as:

- `seeds/`
- `migrations/`

Do not use runtime `app/attendance.db` as the seed database.

## Changes made

### Tests are no longer ignored

Removed this rule from `.gitignore`:

```text
tests/
```

Command:

```bash
rtk proxy git check-ignore -v tests/
```

Result:

```text
exit code 1, no output
```

This means `tests/` is not ignored and can be added to Git.

### Runtime data remains ignored

Command:

```bash
rtk proxy git check-ignore -v --no-index app/attendance.db data/embeddings/db.pkl backups/
```

Result:

```text
.gitignore:36:*.db	app/attendance.db
.gitignore:29:data/	data/embeddings/db.pkl
.gitignore:30:backups/	backups/
```

### Existing tracked runtime DB was removed from the index

Before this task, `app/attendance.db` was already tracked by Git, so `.gitignore` alone could not stop tracking it.

Command:

```bash
rtk proxy git rm --cached app/attendance.db
```

Result:

```text
rm 'app/attendance.db'
```

The local file remains on disk. The Git index now records it as removed so a future commit can stop tracking it.
